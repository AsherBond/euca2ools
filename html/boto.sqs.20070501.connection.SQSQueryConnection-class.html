receive_message(self, queue_url, number_messages=1, visibility_timeout=None, message_class=<class boto.sqs.message.Message at 0x1168470>)

source code 
This provides the same functionality as the read and get_messages methods of the queue object. The only reason this is included here is that there is currently a bug in SQS that makes it impossible to read a message from a queue owned by someone else (even if you have been granted appropriate permissions) via the REST interface. As it turns out, I need to be able to do this so until the REST interface gets fixed this is the workaround.
